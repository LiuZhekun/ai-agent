# 前端开发指南

## 目的

面向后端开发人员说明前端本地开发、生产构建与联调关键参数，便于快速完成接口联调和问题定位。

## 前置条件

- Node.js 18+
- npm 9+（或 pnpm/yarn，以下命令以 npm 为例）
- 已启动后端 API（HTTP + SSE）

## 安装依赖与启动

### 开发环境

```bash
cd ai-agent-ui
npm install
npm run dev
```

默认开发地址通常为 `http://localhost:5173`（以实际 Vite 配置为准）。

### 生产构建与运行

```bash
cd ai-agent-ui
npm run build
npm run preview
```

`npm run preview` 用于本地验证生产包，默认端口通常为 `4173`。

## 与后端联调参数说明

后端联调建议至少确认以下参数一致：

- `API Base`：HTTP 接口基础地址，例如 `http://127.0.0.1:8080/api`。
- `SSE Path`：SSE 流地址，例如 `/chat/stream`，最终请求地址为 `${API_BASE}${SSE_PATH}`。
- `SSE Timeout`：前端等待超时（如 60s/120s）；超时后应自动中断并提示“可重试”。
- `Reconnect Delay`：SSE 断线重连间隔（建议 1000~3000ms，支持指数退避）。
- `eventId 去重`：基于服务端返回的 `id` 或 `eventId` 做幂等消费，避免重连后重复渲染。

推荐处理流程：

1. 建立 SSE 连接后，记录最新 `eventId`。
2. 接收事件时先判断 `eventId` 是否已处理，已处理则忽略。
3. 连接中断后带上 `Last-Event-ID`（或本地保存的 `eventId`）重连。
4. 若重连失败达到阈值，提示用户手动重试，并保留上下文。

## 常见环境变量

常见变量示例（以 Vite 前缀 `VITE_` 为例）：

```bash
VITE_API_BASE=http://127.0.0.1:8080/api
VITE_SSE_PATH=/chat/stream
VITE_SSE_TIMEOUT_MS=120000
VITE_SSE_RECONNECT_MS=1500
VITE_ENABLE_EVENT_DEDUP=true
```

说明：

- 本地开发一般放在 `.env.local`。
- 生产环境建议通过 CI/CD 注入，避免硬编码到仓库。

## 代理配置示例

如前端与后端同机开发，可通过 Vite 代理规避跨域：

```ts
// vite.config.ts
import { defineConfig } from "vite";

export default defineConfig({
  server: {
    proxy: {
      "/api": {
        target: "http://127.0.0.1:8080",
        changeOrigin: true
      },
      "/chat/stream": {
        target: "http://127.0.0.1:8080",
        changeOrigin: true
      }
    }
  }
});
```

如果网关统一转发，也可仅保留 `/api` 代理，SSE 走同域路径。

## SSE 事件建议处理

- `heartbeat`：刷新连接保活时间。
- `tool_trace`：写入时间线。
- `final_answer`：更新最终文本。
- `error`：展示错误提示并保留上下文。

## 常见问题

- 接口 `404/502`：优先检查 `VITE_API_BASE` 与代理 target 是否一致。
- SSE 无消息：确认后端响应头为 `text/event-stream`，并检查反向代理是否关闭缓冲。
- SSE 重复渲染：检查 `eventId` 去重逻辑和重连时的 `Last-Event-ID`。
- Markdown 渲染异常：检查输入是否包含未闭合标签。
- 图表不显示：确认后端返回的是合法 ECharts option JSON。
