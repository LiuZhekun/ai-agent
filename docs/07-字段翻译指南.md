# 字段翻译指南

## 目的

将自然语言输入中的“显示值”映射为业务系统所需“存储值”。

## 前置条件

- 已引入 `ai-agent-translator`
- 业务表中存在字典或可查询映射表

## 接入步骤

1. 在 DTO 字段上标注 `@TranslateField`。
2. 配置冲突策略（`USER_SELECT/FIRST_MATCH/FAIL`）。
3. 为未命中场景准备兜底提示。
4. 通过测试覆盖翻译成功、多结果、未命中三类情况。

## 示例代码

```java
@TranslateField(type = "ENTITY_REF", source = "deptName", target = "sys_department", lookupField = "name", resultField = "id")
private String deptName;
```

## 示例命令

```bash
mvn -pl ai-agent-translator test
```

## 常见问题

- 多结果冲突：优先走 `USER_SELECT`，避免误写入。
- SQL 注入风险：仅允许白名单表和字段参与翻译查询。
