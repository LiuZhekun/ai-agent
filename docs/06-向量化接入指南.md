# 向量化接入指南

## 一、概述

向量化模块（`ai-agent-vectorizer`）能将业务数据库中的结构化数据自动转换为向量并写入 Milvus 向量数据库，从而支撑 **语义检索** 和 **知识增强（RAG）** 等 AI 场景。

**核心流程：** 数据库表数据 → 文本拼装 → Embedding 向量化 → 写入 Milvus → 语义召回

```
┌──────────────┐    ┌───────────────┐    ┌───────────────┐    ┌──────────┐
│  MySQL 业务表 │───▶│ 文本拼装/模板化 │───▶│ Embedding 模型 │───▶│  Milvus  │
└──────────────┘    └───────────────┘    └───────────────┘    └──────────┘
       ▲                                                           │
       │            @VectorIndexed 注解驱动                          │
       │                                                           ▼
  业务数据写入                                               语义相似度检索
```

## 二、前置条件

| 依赖项 | 说明 |
|--------|------|
| Milvus | 向量数据库，需 v2.x+ 可用（默认 `127.0.0.1:19530`） |
| MySQL | 存储业务数据的关系型数据库 |
| Embedding 模型 | 通过 Spring AI 配置的嵌入模型（如 DashScope、OpenAI 等） |

确保已部署中间件，可参考 [中间件部署手册](03-中间件部署手册.md)。

## 三、接入步骤

### 3.1 引入 Maven 依赖

在业务模块的 `pom.xml` 中添加：

```xml
<dependency>
    <groupId>io.github.aiagent</groupId>
    <artifactId>ai-agent-vectorizer</artifactId>
    <version>${project.version}</version>
</dependency>
```

### 3.2 配置 Milvus 连接

在 `application.yml` 中配置向量数据库连接：

```yaml
spring:
  ai:
    vectorstore:
      milvus:
        client:
          host: ${MILVUS_HOST:127.0.0.1}
          port: ${MILVUS_PORT:19530}
        database-name: ${MILVUS_DATABASE:default}
        collection-name: ${MILVUS_COLLECTION:ai_agent_knowledge}
        embedding-dimension: ${MILVUS_EMBEDDING_DIMENSION:1024}   # 需与 Embedding 模型输出维度一致
        index-type: ${MILVUS_INDEX_TYPE:IVF_FLAT}                 # 可选：IVF_FLAT / HNSW / FLAT
        metric-type: ${MILVUS_METRIC_TYPE:COSINE}                 # 可选：COSINE / L2 / IP
        initialize-schema: true                                    # 首次启动设为 true，自动创建 Collection
```

> **注意：** `embedding-dimension` 必须与你使用的 Embedding 模型输出维度匹配。例如 DashScope text-embedding-v2 输出 1536 维，则此处应改为 1536。

### 3.3 配置向量同步参数

```yaml
ai:
  agent:
    vector:
      sync:
        enabled: true                        # 是否启用向量同步（生产环境按需开关）
        cron-expression: "0 */5 * * * *"     # 增量同步定时任务，默认每 5 分钟执行一次
        batch-size: 200                      # 批量写入大小
```

### 3.4 在实体类上标注 `@VectorIndexed`

在需要向量化的实体类上添加注解，声明哪些字段参与向量化：

```java
import com.baomidou.mybatisplus.annotation.TableName;
import io.github.aiagent.vectorizer.annotation.VectorIndexed;

@TableName("sys_user")
@VectorIndexed(
    collection = "demo_user_profile",
    fields = {"name", "phone", "email", "gender"},
    textTemplate = "姓名:{name}\n手机号:{phone}\n邮箱:{email}\n性别:{gender}"
)
public class User {
    private Long id;
    private String name;
    private String phone;
    private String email;
    private String gender;
    // ... 其他字段和 getter/setter
}
```

**`@VectorIndexed` 注解参数说明：**

| 参数 | 必填 | 说明 |
|------|------|------|
| `collection` | 是 | Milvus 中的 Collection 名称，建议按业务域命名，如 `order_info`、`product_catalog` |
| `fields` | 是 | 参与向量化的实体字段名（Java 属性名），这些字段的值会被拼装成文本后进行 Embedding |
| `textTemplate` | 否 | 文本拼装模板，支持 `{fieldName}` 占位符。为空时按 `字段名: 值` 格式自动拼接 |

**文本模板示例对比：**

```
# 使用自定义模板（推荐，语义更清晰）
textTemplate = "姓名:{name}\n手机号:{phone}\n邮箱:{email}\n性别:{gender}"
→ 生成文本: "姓名:张三\n手机号:13800138000\n邮箱:zhangsan@example.com\n性别:男"

# 不设置模板（自动拼接）
textTemplate = ""  (默认)
→ 生成文本: "name: 张三\nphone: 13800138000\nemail: zhangsan@example.com\ngender: 男"
```

> **最佳实践：** 建议使用自定义 `textTemplate` 并添加中文描述性前缀，这样能提升语义检索的准确率。

### 3.5 注册实体提供者

实现 `VectorSyncEntityProvider` 接口，告诉同步框架需要处理哪些实体：

```java
import io.github.aiagent.vectorizer.VectorSyncEntityProvider;
import org.springframework.stereotype.Component;
import java.util.List;

@Component
public class MyVectorSyncEntityProvider implements VectorSyncEntityProvider {

    @Override
    public List<Class<?>> entities() {
        // 返回所有需要向量同步的实体类
        return List.of(
            User.class
            // Product.class,    // 如需同步更多实体，在此添加
            // Order.class
        );
    }
}
```

### 3.6 触发首次全量同步

启动应用后，调用 REST 接口执行一次全量同步，将存量数据写入向量库：

```bash
curl -X POST http://127.0.0.1:8080/api/agent/vector/sync
```

返回示例：

```json
{"status": "ok"}
```

首次同步完成后，定时任务会按配置的 cron 表达式（默认每 5 分钟）自动执行增量同步。

## 四、同步机制详解

### 4.1 全量同步 vs 增量同步

| 类型 | 触发方式 | 数据范围 | 适用场景 |
|------|----------|----------|----------|
| 全量同步 | REST 接口手动触发 | 表中所有数据 | 首次接入、数据修复 |
| 增量同步 | 定时任务自动执行 | 上次同步后变更的数据 | 日常运行 |

### 4.2 增量同步的时间字段

增量同步通过表中的更新时间字段判断哪些数据发生了变更。系统会按以下顺序自动探测时间列：

`updated_at` → `update_time` → `gmt_modified` → `modified_at` → `updatedAt` → `updateTime`

如果表中不存在以上任何一个字段，增量同步会**自动退化为全量同步**。

> **建议：** 确保业务表中包含 `update_time` 或 `updated_at` 字段，并在数据更新时维护该字段，以充分利用增量同步的性能优势。

### 4.3 表名映射规则

实体类名会按驼峰转下划线规则映射为数据库表名：

| 实体类名 | 映射表名 |
|----------|----------|
| `User` | `user` |
| `SysUser` | `sys_user` |
| `ProductCategory` | `product_category` |

如果使用了 `@TableName` 注解指定表名，实际查库仍以驼峰转换后的类名为准（这是当前实现的行为）。请确保实体类名与表名的转换关系一致。

## 五、查询与检索

### 5.1 通过 Agent 对话进行语义检索

数据写入向量库后，Agent 在对话过程中会自动进行语义召回。相关配置：

```yaml
ai:
  agent:
    memory:
      vector-top-k: 5       # 每次语义召回的最大文档数
      vector-threshold: 0.7  # 相似度阈值（0~1），低于此值的结果将被过滤
```

当用户提问 "帮我查一下张三的联系方式" 时，Agent 会将问题向量化后在 Milvus 中检索语义相似的文档，找到匹配的用户资料并作为上下文辅助回答。

### 5.2 自定义向量检索（进阶）

如需在业务代码中直接进行向量检索，可以注入 Spring AI 的 `VectorStore` 使用：

```java
import org.springframework.ai.vectorstore.VectorStore;
import org.springframework.ai.vectorstore.SearchRequest;
import org.springframework.ai.document.Document;
import java.util.List;

@Service
public class MySearchService {

    private final VectorStore vectorStore;

    public MySearchService(VectorStore vectorStore) {
        this.vectorStore = vectorStore;
    }

    public List<Document> search(String query) {
        SearchRequest request = SearchRequest.builder()
            .query(query)
            .topK(5)
            .similarityThreshold(0.7)
            .build();
        return vectorStore.similaritySearch(request);
    }
}
```

### 5.3 实现 VectorKnowledgeBase 接口（可选）

框架核心模块定义了 `VectorKnowledgeBase` 接口用于长期记忆检索。如需将向量检索接入 Agent 的记忆系统，可自行实现该接口：

```java
import io.github.aiagent.core.memory.VectorKnowledgeBase;
import org.springframework.ai.vectorstore.VectorStore;
import org.springframework.ai.vectorstore.SearchRequest;
import org.springframework.ai.document.Document;
import org.springframework.stereotype.Component;
import java.util.List;

@Component
public class MilvusVectorKnowledgeBase implements VectorKnowledgeBase {

    private final VectorStore vectorStore;

    public MilvusVectorKnowledgeBase(VectorStore vectorStore) {
        this.vectorStore = vectorStore;
    }

    @Override
    public List<String> search(String query, int topK, double threshold) {
        SearchRequest request = SearchRequest.builder()
            .query(query)
            .topK(topK)
            .similarityThreshold(threshold)
            .build();
        return vectorStore.similaritySearch(request)
            .stream()
            .map(Document::getText)
            .toList();
    }
}
```

注册该 Bean 后，`AgentMemoryManager.searchKnowledge()` 即可自动调用向量检索。

## 七、文档解析（预留能力）

模块内置了文档解析器，可将非结构化文档转为文本后参与向量化：

| 文档类型 | 解析器 | 解析策略 |
|----------|--------|----------|
| PDF | `PdfDocumentParser` | 按页提取文本（基于 Apache PDFBox） |
| Word (.docx) | `WordDocumentParser` | 按段落提取文本（基于 Apache POI） |
| Excel (.xlsx) | `ExcelDocumentParser` | 按行转为 `列名:值` 格式文本（基于 Apache POI） |

> **注意：** 当前版本中文档解析器为预留扩展能力，主同步链路仅处理数据库表数据。

## 九、常见问题

### Q1: 同步执行很慢怎么办？

- 检查 Embedding 模型的网络延迟，确保 API 调用稳定
- 观察 Milvus 写入吞吐，必要时增加 Milvus 资源配置
- 调小 `batch-size` 减少单批压力

### Q2: 语义检索命中率低？

- 优化 `textTemplate`，添加业务关键词提升语义表达质量
- 适当降低 `vector-threshold`（如从 0.7 降到 0.5）扩大召回范围
- 检查 `fields` 是否包含了足够的语义信息字段

### Q3: 增量同步没有生效？

- 确认表中存在更新时间字段（`update_time`、`updated_at` 等）
- 确认业务代码在数据更新时正确维护了更新时间字段
- 检查日志中是否出现 "Fallback to full sync" 提示

### Q4: 启动时报 Milvus 连接失败？

- 如果不需要向量检索功能，可设置 `ai.agent.vector.sync.enabled=false` 关闭同步
- 确认 Milvus 服务已正常启动，端口 19530 可访问
- 首次使用确保 `initialize-schema: true` 以自动创建 Collection

### Q5: 如何查看 Milvus 中已写入的向量数据？

- 推荐使用 [Attu](https://github.com/zilliztech/attu)（Milvus 官方可视化管理工具）连接查看
- 也可以通过 Milvus SDK 的 `query` 接口直接查询
