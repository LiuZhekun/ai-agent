# 中间件部署手册

## 目的

说明 MySQL/Redis/Milvus/Nginx 的部署方式，覆盖在线与离线场景。

## 前置条件

- Linux 优先（Ubuntu/CentOS）
- 可用的 Docker 环境
- 已准备 `deploy/.env`

## 建议虚拟机规格

以下规格按 MySQL + Redis + Milvus + Nginx + 后端服务在同机部署估算。

### 最低配置（单机演示可跑通）

- CPU：4 vCPU
- 内存：8 GB
- 磁盘：80 GB SSD
- 网络：100 Mbps（内网）

适用场景：功能演示、个人验证。  
风险提示：Milvus 建索引和批量写入时可能出现明显抖动。

### 推荐配置（多人联调）

- CPU：8 vCPU
- 内存：16 GB
- 磁盘：150 GB SSD
- 网络：1 Gbps（内网优先）

适用场景：3-10 人并行联调、持续写入测试、较稳定的响应时延。

### 生产建议（中小规模）

- CPU：16 vCPU 及以上
- 内存：32 GB 及以上
- 磁盘：300 GB NVMe SSD（建议拆分数据盘/日志盘）
- 网络：1 Gbps 起，跨可用区建议专线或低时延 VPC 网络

补充建议：

- MySQL 与 Milvus 优先使用独立存储卷，避免容器重建导致数据丢失。
- 日志建议接入集中式采集（如 ELK/Loki），避免单机磁盘被日志打满。
- 生产建议将 MySQL、Milvus 与应用层拆分部署，降低资源争用。

## 在线部署步骤

```bash
cd deploy
cp -n .env.example .env
./install-all.sh
```

安装脚本会执行：
1. 检测并安装 Docker/Compose（Ubuntu/CentOS）
2. 启动编排
3. 初始化数据库
4. 健康检查并输出访问地址

## 离线部署步骤

1. 在线机器执行 `deploy/offline/export-images.sh`
2. 传输 `images/ai-agent-images.tar`
3. 离线机器执行 `deploy/offline/import-images.sh`
4. 离线机器执行 `deploy/install-all.sh`

## 示例命令

```bash
docker compose ps
docker compose logs -f mysql
docker compose logs -f backend
```

## 常见问题

- Milvus 启动慢：首次拉起耗时较长，建议等待并查看日志。
- MySQL 初始化未生效：检查 `deploy/init-db` 目录及 `.env` 的库名配置。

## 单机演示与多人联调差异建议

- 单机演示：优先保证“可用”，可将全部组件放同一台机器，降低运维复杂度。
- 多人联调：建议至少拆分“数据库/向量库”和“应用层”，减少资源争抢。
- 单机演示：SSE 和接口超时可适当放宽，便于定位问题。
- 多人联调：建议开启更严格的连接池和请求限流，防止突发请求拖垮实例。
- 单机演示：可接受短时服务波动。
- 多人联调：建议配置基础监控告警（CPU、内存、磁盘 IO、容器重启次数）。
