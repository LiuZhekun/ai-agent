# 工具开发指南

## 目的

规范工具开发方式，确保工具可被模型稳定调用并具备安全边界。

## 前置条件

- 已引入 `ai-agent-core` 或 starter
- 了解 Spring Bean 与 `@Tool` 注解

## 开发步骤

1. 创建工具类并标注 `@AgentTool`。
2. 在方法上标注 `@Tool`，命名应语义清晰。
3. 入参优先使用 DTO，避免多个松散参数。
4. 对写操作配置确认流程与风险等级。
5. 编写单元测试覆盖成功/失败分支。

## 示例代码

```java
@AgentTool(name = "user-tools", description = "用户查询工具")
public class UserTools {
    @Tool(name = "queryUser")
    public List<UserVO> queryUser(String name) { ... }
}
```

## 示例命令

```bash
mvn -pl ai-agent-core test
```

## 常见问题

- 工具未被识别：确认类被 Spring 扫描、方法有 `@Tool`。
- 返回结果太乱：在工具层做结构化返回，避免大段拼接字符串。
