# 架构设计

## 目的

描述系统分层、调用链路与模块依赖，指导扩展开发。

## 前置条件

- 已了解模块目录结构
- 具备 Spring Boot 自动装配基础

## 分层架构

1. **核心层**：`core` 负责会话、工具、记忆、事件模型。
2. **能力层**：`knowledge/vectorizer/translator` 提供可选增强能力。
3. **组装层**：`starter` 进行条件装配并暴露 Web API。
4. **应用层**：`demo/ui` 承载业务与交互。
5. **基础设施层**：`deploy` 提供运行环境。

## 关键流程（对话）

1. 前端发起 `chat` 请求。
2. `AgentEngine` 获取会话并执行 Advisor 链。
3. 组装系统 Prompt（含知识上下文）。
4. 调用 LLM/工具，产生 SSE 事件流。
5. 前端按事件类型更新消息、图表与轨迹。

## L3 RAG 位置

- 位于 `knowledge/rag` 包。
- 索引来源为 L0 片段。
- 请求时按 query 进行检索、重排并注入带引用来源的上下文。

## 示例命令

```bash
mvn -pl ai-agent-core,ai-agent-knowledge -am test
```

## 常见问题

- 需要新增能力模块：优先通过 `starter` 的条件装配接入，避免强耦合到 `core`。
