# 快速开始（从零到全量功能体验）

> 本文面向**首次接触本项目的新人**，手把手带你在 30 分钟内完成启动并体验全部核心功能。
> 只需复制粘贴命令，无需理解内部原理。

---

## 最终效果

完成本指南后你将拥有：

- 一套运行中的中间件（MySQL + Redis + Milvus）
- 一个可对话的后端 AI Agent 服务
- 一个浏览器可访问的前端聊天界面
- 并能体验以下全部功能：流式对话、工具调用轨迹、思考摘要、澄清对话、写操作确认、向量语义检索

## 第 0 步：环境准备（约 10 分钟）

### 需要安装的软件

| 软件 | 最低版本 | 检查命令 | 安装参考 |
|------|---------|---------|---------|
| JDK | 17 | `java -version` | [Adoptium](https://adoptium.net/) |
| Maven | 3.9 | `mvn -v` | [Maven 官网](https://maven.apache.org/download.cgi) |
| Node.js | 18 | `node -v` | [Node.js 官网](https://nodejs.org/) |
| Docker | 24 | `docker -v` | [Docker Desktop](https://www.docker.com/products/docker-desktop/) |

> **Windows 用户**：Docker Desktop 安装后需确认已启动（系统托盘有鲸鱼图标）。

### 逐项检查（复制到终端执行）

**Windows PowerShell**：

```powershell
java -version
mvn -v
node -v
docker -v
docker compose version
```

**macOS / Linux**：

```bash
java -version
mvn -v
node -v
docker -v
docker compose version
```

> 全部命令无报错即可继续。如果某一项不通过，请先安装对应软件。

### 获取 LLM API Key（二选一）

| 提供商 | 注册地址 | 环境变量名 | 推荐模型 |
|--------|---------|-----------|---------|
| 阿里 DashScope | [控制台](https://dashscope.console.aliyun.com/) | `DASHSCOPE_API_KEY` | qwen-plus |
| DeepSeek | [开放平台](https://platform.deepseek.com/) | `DEEPSEEK_API_KEY` | deepseek-chat |

> 推荐 DashScope，注册即送免费额度，无需充值即可体验。

## 第 1 步：克隆项目

```bash
git clone <项目仓库地址>
cd ai-agent
```

> 如果你已有项目代码，跳过此步，直接 `cd` 进入项目根目录。

---

## 第 2 步：启动中间件（MySQL + Redis + Milvus）

中间件通过 Docker 一键启动，数据库表和示例数据会自动初始化。

### Windows PowerShell

```powershell
cd deploy
docker compose --env-file .env.example up -d mysql redis milvus
```

### macOS / Linux

```bash
cd deploy
docker compose --env-file .env.example up -d mysql redis milvus
```

### 验证中间件是否启动成功

等待约 30 秒后执行：

```bash
docker compose --env-file .env.example ps
```

**预期输出**：三个容器状态均为 `Up (healthy)`。

```
NAME               STATUS
ai-agent-mysql     Up (healthy)
ai-agent-redis     Up (healthy)
ai-agent-milvus    Up (healthy)
```

> Milvus 首次启动较慢（30-90 秒），状态可能暂时显示 `Up (health: starting)`，等待变为 `healthy` 即可。

> **首次拉取镜像可能需要 5-15 分钟**，取决于网络速度，请耐心等待。
> 如果拉取超时，可配置 Docker 镜像加速器（搜索"Docker 镜像加速"）。

### 中间件端口一览

| 服务 | 默认端口 | 用途 |
|------|---------|------|
| MySQL | 3306 | 业务数据存储 |
| Redis | 6379 | 会话管理与短期记忆 |
| Milvus | 19530 | 向量检索（RAG） |

## 第 3 步：设置环境变量（LLM API Key）

根据你选择的 LLM 提供商，设置对应环境变量。

### 方案 A：DashScope（推荐）

**Windows PowerShell**：

```powershell
$env:DASHSCOPE_API_KEY="sk-你的DashScope密钥"
```

**macOS / Linux**：

```bash
export DASHSCOPE_API_KEY="sk-你的DashScope密钥"
```

### 方案 B：DeepSeek

**Windows PowerShell**：

```powershell
$env:DEEPSEEK_API_KEY="sk-你的DeepSeek密钥"
```

**macOS / Linux**：

```bash
export DEEPSEEK_API_KEY="sk-你的DeepSeek密钥"
```

> **注意**：环境变量仅在当前终端窗口有效。关闭终端后需要重新设置。
> 如需永久生效，Windows 可在"系统属性 → 环境变量"中添加，Linux/macOS 可写入 `~/.bashrc` 或 `~/.zshrc`。

## 第 4 步：启动后端服务

回到项目根目录，执行以下命令。

### 方案 A：使用 DashScope

**Windows PowerShell**：

```powershell
cd ..
mvn -pl ai-agent-demo -am spring-boot:run "-Dspring-boot.run.profiles=dashscope"
```

**macOS / Linux**：

```bash
cd ..
mvn -pl ai-agent-demo -am spring-boot:run -Dspring-boot.run.profiles=dashscope
```

### 方案 B：使用 DeepSeek

**Windows PowerShell**：

```powershell
cd ..
mvn -pl ai-agent-demo -am spring-boot:run "-Dspring-boot.run.profiles=deepseek"
```

**macOS / Linux**：

```bash
cd ..
mvn -pl ai-agent-demo -am spring-boot:run -Dspring-boot.run.profiles=deepseek
```

### 如何判断启动成功？

观察终端输出，当看到类似以下日志时说明启动成功：

```
Started AiAgentDemoApplication in X.XXX seconds
```

> **首次编译会下载依赖**，约需 3-10 分钟。如果 Maven 下载慢，建议配置阿里云镜像。
> 在 Maven 的 `settings.xml`（通常在 `~/.m2/settings.xml`）中添加：
>
> ```xml
> <mirror>
>   <id>aliyunmaven</id>
>   <mirrorOf>*</mirrorOf>
>   <name>阿里云公共仓库</name>
>   <url>https://maven.aliyun.com/repository/public</url>
> </mirror>
> ```

## 第 6 步：验证后端服务是否正常

**打开第三个终端窗口**，用以下命令测试接口。

### 测试 1：检查服务存活

```bash
curl http://127.0.0.1:8080/api/agent/sessions
```

**预期返回**：包含 `count` 字段的 JSON，如 `{"count":0,"sessions":[]}`。

### 测试 2：流式对话

**Windows PowerShell**：

```powershell
curl -N -X POST "http://127.0.0.1:8080/api/agent/chat" `
  -H "Content-Type: application/json" `
  -d '{"sessionId":"quickstart-1","message":"查询所有用户"}'
```

**macOS / Linux**：

```bash
curl -N -X POST "http://127.0.0.1:8080/api/agent/chat" \
  -H "Content-Type: application/json" \
  -d '{"sessionId":"quickstart-1","message":"查询所有用户"}'
```

**预期返回**：多条 `data:` 开头的 SSE 事件，包括 `THINKING_SUMMARY`、`TOOL_TRACE`、`FINAL_ANSWER`、`COMPLETED` 等。

> 如果返回 404 或连接失败，请确认后端已启动且端口为 8080。

## 第 7 步：全量功能体验

在浏览器前端聊天窗口中，按顺序输入以下内容体验全部功能。

### 7.1 工具调用 + 轨迹展示

在输入框输入：

```
查询技术部用户
```

**预期效果**：
- 界面出现"工具调用轨迹"（显示 Agent 调用了哪些工具、参数和结果）
- 最终返回技术部的用户列表（张三、李四、王伟、陈晨）

### 7.2 澄清对话（AI 主动追问）

输入：

```
帮我新增一个用户
```

**预期效果**：
- AI 不会直接执行，而是追问你："请提供用户姓名、手机号、部门等信息"
- 这就是"澄清对话"功能——当参数不完整时，AI 会主动索要缺失信息

### 7.3 写操作确认（安全拦截）

输入：

```
把李四的手机号改成 13800000000
```

**预期效果**：
- 界面弹出"执行确认"提示，显示即将执行的修改操作
- 需要你点击"确认"后才会真正执行
- 这就是"写操作确认"功能——所有写操作默认需要人工审批

### 7.4 向量语义检索（RAG 能力）

**步骤一：先触发向量同步**

在第三个终端窗口执行（或用浏览器访问）：

```bash
curl -X POST http://127.0.0.1:8080/api/agent/vector/sync
```

> 这一步会把数据库中的业务数据同步到 Milvus 向量库，首次同步需要等待几秒。

**步骤二：回到前端，输入**

```
按语义帮我找和技术支持相关的内容
```

**预期效果**：
- AI 通过向量相似度搜索找到语义相关的内容并返回
- 这就是 RAG（检索增强生成）能力

## 附录 B：配置文件说明

项目有 3 个配置文件，按以下结构组织：

```
ai-agent-demo/src/main/resources/
├── application.yml              ← 通用配置（端口、数据库、Redis、向量库、Agent 行为）
├── application-dashscope.yml    ← DashScope 专用（API Key、地址、模型）
└── application-deepseek.yml     ← DeepSeek 专用（API Key、地址、模型）
```

- **`application.yml`**：不区分 LLM 提供商的通用配置。包括服务端口、MySQL/Redis/Milvus 连接信息、Agent 工具调用/记忆/安全/澄清/SSE 等行为参数。**一般无需修改**，除非你的中间件端口或密码不是默认值。
- **`application-dashscope.yml`**：仅在 `--spring.profiles.active=dashscope` 时生效，配置 DashScope 的 API Key、接入地址和模型名。
- **`application-deepseek.yml`**：仅在 `--spring.profiles.active=deepseek` 时生效，配置 DeepSeek 的 API Key、接入地址和模型名。

> 所有敏感信息（密钥、密码）均通过 `${ENV_VAR:默认值}` 方式读取环境变量，**不要直接把密钥写死到配置文件中**。

## 常见问题排查

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 后端接口返回 404 | 后端未启动或端口不对 | 确认终端有 `Started AiAgentDemoApplication` 日志，端口默认 8080 |
| 前端页面空白或不流式 | 后端未正确返回 SSE | 检查后端日志是否有报错，确认 API Key 已正确设置 |
| 提示 "Session Busy" | 同一会话正在处理上一条消息 | 等上一轮回复结束，或换一个 sessionId |
| 模型返回空 / 报错 | API Key 无效或未设置 | 重新检查环境变量，确保 Key 有余额 |
| Milvus 连接失败 | Docker 中 Milvus 未启动 | 执行 `docker compose ps` 确认 milvus 状态为 Up |
| MySQL 连接被拒 | Docker 中 MySQL 未就绪 | MySQL 首次启动初始化约需 30 秒，稍等后重试 |
| Maven 编译报错 | JDK 版本不对 | 执行 `java -version` 确认是 17 或以上 |
| npm install 失败 | Node.js 版本过低 | 执行 `node -v` 确认是 18 或以上 |
| Docker 拉取镜像超时 | 网络不通或未配加速器 | 配置 Docker 镜像加速器后重试 |
| Windows mvn 命令参数报错 | PowerShell 对 `-D` 参数解析问题 | 用双引号包裹参数：`"-Dspring-boot.run.profiles=dashscope"` |

## 完整启动命令速查

以下是从零启动全部服务的完整命令序列（以 DashScope + Windows PowerShell 为例）：

```powershell
# 1. 进入项目
cd d:\workSpace\ai\ai-agent

# 2. 启动中间件
cd deploy
docker compose --env-file .env.example up -d mysql redis milvus
cd ..

# 3. 设置 API Key
$env:DASHSCOPE_API_KEY="sk-你的密钥"

# 4. 启动后端（此终端会被占用，后续步骤请开新终端）
mvn -pl ai-agent-demo -am spring-boot:run "-Dspring-boot.run.profiles=dashscope"
```

```powershell
# 5. 新终端：启动前端
cd d:\workSpace\ai\ai-agent\ai-agent-ui
npm install
npm run dev
```

```powershell
# 6. 新终端：触发向量同步（可选，体验 RAG 时再执行）
curl -X POST http://127.0.0.1:8080/api/agent/vector/sync
```

浏览器打开 **http://localhost:5173/** 开始对话。
