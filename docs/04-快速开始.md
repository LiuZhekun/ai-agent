# 快速开始

## 目的

让你在最短路径内跑通一套可对话的 AI Agent 服务。

## 前置条件

- 完成 `02-开发环境搭建.md`
- 已获取可用 LLM API Key
- 已了解本项目默认使用 OpenAI 兼容层（通过不同 profile 切换提供商）

## 步骤

1. **启动服务栈**
   - Linux/macOS：执行 `cd deploy && ./install-all.sh`
   - Windows（PowerShell）：执行 `cd deploy` 后运行 `docker compose --env-file .env.example up -d mysql redis milvus`
2. **启动后端**
   - 默认启动（使用 `application.yml`）：`mvn -pl ai-agent-demo -am spring-boot:run`
   - 使用 DashScope：`mvn -pl ai-agent-demo -am spring-boot:run -Dspring-boot.run.profiles=dashscope`
   - 使用 DeepSeek：`mvn -pl ai-agent-demo -am spring-boot:run -Dspring-boot.run.profiles=deepseek`
3. **启动前端**
   - 执行 `cd ai-agent-ui && npm install && npm run dev`
4. **发起对话**
   - 浏览器打开前端页面，输入“查询技术部用户”

## 模型与配置说明

- 项目统一使用 `spring.ai.openai.*` 兼容配置，通过 profile 切换不同模型提供商。
- `application-dashscope.yml`：配置 DashScope 兼容地址与模型。
- `application-deepseek.yml`：配置 DeepSeek 兼容地址与模型。
- 通用业务开关在 `application.yml`：
  - `ai.agent.llm.provider=openai-compatible`
  - `ai.agent.tool.timeout-seconds=10`
  - `ai.agent.safety.write-need-confirm=true`

## IDEA 启动时指定模型

1. 打开 `Run/Debug Configurations`，选择后端启动项（`ai-agent-demo`）。
2. 在 `Active profiles` 填写 `dashscope` 或 `deepseek`。
3. 如需临时覆盖模型，在 `Program arguments` 添加：
   - `--spring.ai.openai.chat.options.model=qwen-plus`
   - 或 `--spring.ai.openai.chat.options.model=deepseek-chat`
4. 在 `Environment variables` 配置密钥（建议）：
   - DashScope：`DASHSCOPE_API_KEY=...`
   - DeepSeek：`DEEPSEEK_API_KEY=...`

## 示例命令

```bash
curl -N -X POST "http://127.0.0.1:8080/api/agent/chat" \
  -H "Content-Type: application/json" \
  -d "{\"sessionId\":\"quickstart-1\",\"message\":\"查询所有用户\"}"
```

## 常见问题

- 接口 404：确认后端是否启动在 `8080`。
- 前端无流式消息：检查网络代理和后端 SSE 日志。
- 模型未生效：检查 `Active profiles` 是否与目标配置文件一致（`dashscope`/`deepseek`）。
- API Key 无效：确认环境变量已在当前运行配置中生效，避免将明文密钥写入仓库。
