# 可观测与测试指南

## 目的

规范系统监控、日志追踪与测试分层策略，确保上线稳定性。

## 前置条件

- 已启用 Spring Boot Actuator（推荐）
- 日志系统支持结构化输出

## 可观测性建议

- 指标：工具调用次数、成功率、耗时、会话时长、SQL 拒绝数。
- 日志：审计日志携带 `traceId/sessionId/toolName`。
- 健康检查：暴露 `/actuator/health` 与关键依赖状态。

## 测试分层

1. 单元测试：核心算法、校验器、翻译策略、裁剪器。
2. 集成测试：数据库访问、工具调用链、SSE 协议。
3. 端到端测试：前后端联调（含澄清与确认流程）。

## 示例命令

```bash
mvn test
mvn -pl ai-agent-core,ai-agent-knowledge test
```

## 常见问题

- 测试不稳定：避免依赖外部实时服务，优先 Mock 或 Testcontainers。
- 线上问题难定位：统一传递 traceId，并在 SSE 事件和日志中保持一致。
